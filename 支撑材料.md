## 一、系统简介


### （一）项目背景

近年来电信网络诈骗犯罪高发，公安机关反诈中心每日接收大量涉诈话术线索，传统人工研判模式面临三大痛点：一是案件量大，逐条人工分析效率低下；二是串并案依赖民警个人经验，不同人员判断标准不一，容易遗漏关联案件；三是诈骗话术翻新速度快，基于关键词规则的匹配方法覆盖面窄，难以应对新型变种。

本项目以中国人民公安大学《融合大语言模型和语义聚类的电信网络诈骗引流话术文本分析》论文为理论基座，将语义聚类方法从单一的话术分类扩展为覆盖全链路的智能研判流程，构建面向实战的AI反诈画像系统。


### （二）系统定位

AI反诈画像系统定位为反诈中心民警的智能辅助研判工具，核心能力包括：

1. 多源数据接入：支持文本直接录入、文件批量上传和OCR图片识别三种方式，适应一线民警多样化的线索采集场景。

2. 自动化研判流水线：6个智能体协同完成数据预处理、智能分案、案件深度分析、团伙聚类发现、画像增强和结果汇总，将原本需要数小时的人工研判压缩至分钟级。

3. 双路径串并案推荐：语义聚类路径基于话术内容相似度发现团伙，Jaccard实体重叠路径基于手机号、银行卡号等实体交叉发现关联案件，两条路径互补，减少遗漏。

4. 可靠性保障：基于客观指标的置信度评估，不依赖大语言模型自评，低置信度结果标记建议人工复核，确保研判结论可信可用。


### （三）技术架构

系统采用三层架构设计：

1. 交互层：基于Vue 3 + Element Plus构建前端界面，集成ECharts图表可视化和vis-network关联网络图谱，通过Nginx反向代理实现静态资源缓存和WebSocket升级。

2. 决策与协同层：以FastAPI为后端框架，ChiefAgent作为中枢编排器调度6阶段智能研判流水线。核心算法包括BGE语义编码（1024维向量）、UMAP降维、HDBSCAN密度聚类、余弦相似度增量聚类和Silhouette+DBI聚类质量评估。

3. 数据与模型层：MySQL存储14张关系型数据表，Redis提供缓存和任务队列，BGE-large-zh-v1.5本地推理引擎保障数据不出域，DeepSeek大语言模型通过云端API提供结构化分析能力。

## 二、系统说明


### （一）核心功能模块

1. 数据预处理模块（PreprocessAgent）

对原始线索进行数据清洗、敏感信息脱敏和质量评分，支持文本、CSV文件和OCR识别结果三种输入。质量评分从完整性、规范性和信息丰富度三个维度评估，低质量数据标记后仍进入后续流程，但影响最终置信度。

2. 智能分案模块（TriageAgent）

基于DeepSeek大语言模型对线索进行诈骗类型分类和紧急程度判定，输出结构化分案结果。置信度基于分类概率分布的集中度和实体提取完整度计算，不依赖LLM自评。低置信度案件标记建议人工复核。

3. 案件分析模块（AnalystAgent）

对分案后的线索进行深度分析，提取关键实体（手机号、银行卡号、社交账号等）、诈骗话术特征和角色分离信息（引流者、实施者、洗钱者），生成结构化分析报告。

4. 团伙聚类模块（ClusterAgent）

采用BGE-large-zh-v1.5将话术文本编码为1024维语义向量，经UMAP降维后由HDBSCAN密度聚类算法发现话术相似的案件簇，识别潜在诈骗团伙。增量聚类机制基于余弦相似度阈值0.75，新线索自动归入已有簇或创建新簇，无需全量重算。聚类质量通过Silhouette轮廓系数和Davies-Bouldin指数评估。

5. 画像增强模块（ProfilerAgent）

对每个聚类簇生成团伙画像，包括作案手法特征、活跃时段、目标人群和威胁评级，输出雷达图用于直观展示团伙特征维度。

6. 结果汇总模块

汇总各阶段输出，生成完整研判报告，支持在线查看和导出。


### （二）串并案推荐机制

系统提供两条互补的串并案路径：

1. 语义聚类路径：基于话术文本的语义相似度，发现使用相似话术模板的关联案件，适用于同一团伙使用固定话术模板的场景。

2. Jaccard实体重叠路径：基于案件中提取的实体（手机号、银行卡号等）计算Jaccard相似度，发现共享关键实体的关联案件，适用于话术不同但涉及相同作案资源的场景。

两条路径结果合并去重后输出，最大限度减少遗漏。


### （三）可靠性保障

1. 置信度评估：基于分类概率集中度、实体提取完整度和聚类质量指标等客观维度计算，不依赖大语言模型的自评输出，避免LLM过度自信问题。

2. 异常回退：LLM调用失败时自动切换至规则引擎，聚类失败时切换至规则分组，保障系统可用性。

3. 人工复核：低置信度结果标记"建议人工复核"，民警可在系统中直接确认或修正，修正结果反馈至后续流程。

## 四、系统原理


### （一）整体设计思路

系统以"数据输入→智能研判→结果输出"为主线，将传统依赖民警经验的研判过程拆解为6个可量化、可追溯的智能体阶段。每个阶段有明确的输入输出定义和异常处理策略，ChiefAgent作为中枢编排器负责阶段调度、进度管理和异常回退。


### （二）语义编码原理

采用BGE-large-zh-v1.5模型将话术文本编码为1024维语义向量。该模型基于Transformer架构，在大规模中文语料上预训练并经过检索任务微调，能够捕捉话术文本的深层语义特征。选择本地推理而非API调用，确保涉案数据不出域，符合公安数据安全要求。编码速度约150ms/条（batch=32），满足实时处理需求。


### （三）聚类算法原理

1. UMAP降维：将1024维语义向量降至低维空间，保留局部和全局流形结构，为密度聚类提供高质量输入。

2. HDBSCAN密度聚类：基于层次密度的聚类算法，无需预设簇数量，能够自动发现任意形状的簇并识别噪声点，相比K-Means更适合话术数据的非球形分布特征。

3. 增量聚类：对新输入的话术向量，计算与已有簇中心的余弦相似度，超过阈值0.75则归入最近簇，否则创建新簇并触发HDBSCAN局部重聚类，避免全量重算的开销。

4. 聚类质量评估：采用Silhouette轮廓系数衡量簇内紧密度和簇间分离度，采用Davies-Bouldin指数衡量簇间重叠程度，两项指标互补评估聚类效果。


### （四）大语言模型应用原理

系统在分案和分析阶段调用DeepSeek大语言模型，通过精心设计的Prompt模板约束输出为结构化JSON格式，避免自由文本解析的不确定性。关键设计包括：

1. 角色分离：在分析阶段要求LLM区分引流者、实施者和洗钱者等不同角色。

2. 置信度客观化：不使用LLM自评置信度，而是基于分类概率分布的熵和实体提取完整度等客观指标计算。

3. 异常回退：LLM调用超时或输出格式异常时，自动切换至规则引擎兜底。


### （五）串并案推荐原理

1. 语义聚类路径：基于话术向量空间中的距离关系，话术模板相似的案件自然聚集为同一簇，揭示使用相同话术模板的作案团伙。

2. Jaccard实体重叠路径：从案件中提取手机号、银行卡号等关键实体，计算案件对之间的Jaccard相似系数（交集/并集），超过阈值则推荐为串并案。该路径能发现话术不同但共享作案资源的关联案件，与语义路径形成互补。
